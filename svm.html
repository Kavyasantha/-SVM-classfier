Inline Question 1

It is possible that once in a while a dimension in the gradcheck will not match exactly. What could such a discrepancy be caused by? Is it a reason for concern? What is a simple example in one dimension where a gradient check could fail? How would change the margin affect of the frequency of this happening? Hint: the SVM loss function is not strictly speaking differentiable

$\color{blue}{\textit Your Answer:}$

Practiclly, this is possible.SVM loss function is defined as max(0, x), where x is is the difference between the scores of incorrect classes and correct class plus delta. For x > 0, we incur a loss, else if x < 0, then we set the loss to 0 as the threshold is set at 0. Max function throws a problem when there is a mismatch between the analytic and numerical gradients. WKT the gradient of the SVM loss function is not defined at the hinge, i.e., at x = 0. Generally, when we have max(x, y), at x = y the gradient is undefined which are ``kinks'' which are the root cause of failed gradchecks.

However, Kinks are not a cause for concern as we can use gradient descent everywhere other than the hinge case where it is undefined. Loss computed at locations where gradient of the function is undefined at that location. However, if that happens, it is safe to just skip that gradient update step when doing gradient descent.

Some examples where gradcheck could fail:

Modulus/absolute function: |x| at x = 0.

ReLU: max(0, z) at z = 0.

Simple example: max(0, x) when x is a very small value, say -1e-5. The analytical gradient would be 0 as the max function outputs 0. However, there can be a gradient mismatch if the numerical gradient can yield a different number if h is greater than x, say h = 1e-4. Increasing the margin (delta) would increase the chances of the class scores being higher (and thus positive) which would mean max(0, x) would output a positive number more often (i.e., the function would tend further away from 0) and thus reduce the frequency of a dimension mismatch during gradcheck which occurs at x = 0.

Inline question 2

Describe what your visualized SVM weights look like, and offer a brief explanation for why they look they way that they do.

$\color{blue}{\textit Your Answer:}$

Description:

  • The visualized SVM weights for each class look like a combination of different images for that class. They appear as rough generalizations of the image features belonging to some group which the subset of weights have identified. They seem like blurred templates for every class.

Interpretation:

  • It can be seen that the SVM has been trained to identify all the distinct features.
  • Because SVM is a linear classifier, it cannot produce a precise decision boundary for all the different forms each image per class may take. For this reason most images of weight visualizations are symmetric - weights are trained to generalize the best. For instance, the horse appears with 2 heads.
